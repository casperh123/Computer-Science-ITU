# Notes lecture 1

## Quality Attribute Scenarios

Basically non-functional requirements. Includes most the 'ties'.

- Availability
- Deployability
- Testability
- Security
- Usability

### The anatomy of a Quality Attribute Scenarios

- Source of stimulus
    - Entity that generated the stimulus (server, human, computer etc.)
- Stimulus
    - The condition that should be considered when it arrives at the system (data, incorrect timing, incorrect response)
- Environment
    - The environment in which the stimulus appears 
- Artifact
    - Some artifact is stimulated. Either entire system or just a piece of it
- Response
    - The action taken after the stimulus arrives
- Response Measure
    - A measure of sorts pertaining to the response. In this case, the response is measureable.
